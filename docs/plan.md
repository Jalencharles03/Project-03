# Project Plan: Brew Haven Website

## Risk Analysis 

### Transfer
- **Risk:** Potential hosting or deployment issues with GitHub Pages  
- **Strategy:** Use GitHub documentation and tutorials to resolve setup issues  

### Avoid
- **Risk:** Overcomplicating the website with features beyond assignment scope  
- **Strategy:** Stick strictly to HTML and CSS and follow the defined scope  

### Mitigate
- **Risk:** Running out of time before the deadline  
- **Strategy:** Start early, break work into smaller tasks, and complete sections incrementally  

- **Risk:** Layout or styling issues  
- **Strategy:** Test frequently in the browser while developing  

### Accept
- **Risk:** Minor visual imperfections or design limitations  
- **Strategy:** Accept small issues as long as functionality and requirements are met  


## Work Breakdown Structure 

### Planning Phase
- Choose project topic (Brew Haven)  
- Create Project Scope Statement (`scope.md`)  

### Design Phase
- Plan layout for homepage (`index.html`)  
- Plan layout for about page (`about.html`)  
- Decide on color scheme and basic styling  

### Development Phase
- Build `index.html` structure  
- Build `about.html` structure  
- Create and link `style.css`  
- Apply consistent styling across pages  

### Testing Phase
- Test navigation links between pages  
- Check layout and readability  
- Fix any errors or broken elements  

### Finalization Phase
- Organize GitHub repository  
- Upload all files (HTML, CSS, documentation)  
- Review assignment requirements  
- Submit final project
