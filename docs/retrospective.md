# Project Retrospective: Brew Haven Website

## What Went Well
One of the biggest successes in this project was staying within the defined scope. By keeping the website limited to two pages and using only HTML and CSS, I was able to complete the project efficiently without overcomplicating it. Planning ahead with the scope statement and project plan also helped keep the work organized and manageable.

Another aspect that went well was the overall structure and consistency of the website. Both pages were clearly organized, navigation worked correctly, and the styling was consistent across the site. Testing along the way helped catch small issues early, which made the final stages smoother.

## What Went Wrong
One challenge during the project was managing time effectively. While the project was not overly complex, it was easy to underestimate how long small tasks like styling and layout adjustments would take. This led to some minor time pressure toward the end of development.

Another issue was occasional trial-and-error with CSS. Some styling choices did not work as expected at first, which required extra time to troubleshoot and adjust. Although this is part of the learning process, it slowed progress at certain points.

## What I Would Do Differently
If I were to approach this project again, I would start the development phase earlier and allocate more time specifically for styling and testing. Breaking tasks into even smaller steps earlier on would help reduce time pressure later in the project.

I would also spend more time planning the visual design before writing code. Creating a clearer layout plan or wireframe at the beginning would reduce the amount of trial-and-error during development.

Overall, I would continue to follow the structured approach from PMUPM, but with more emphasis on early planning and time management to improve efficiency and reduce last-minute adjustments.
